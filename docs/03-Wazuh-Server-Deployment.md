# Chapter 3 : Wazuh Server Deployment

## Overview
This chapter documents the deployment of the Wazuh platform as the central Security Information and Event Management (SIEM) solution within the simulated SME environment. The server was responsible for collecting, indexing and analysing security telemetry generated by Windows and Linux endpoints. The deployment was performed on an Ubuntu Server 24.04 virtual machine running inside UTM on Apple Silicon. During the implementation, several infrastructure challenges were encountered, including storage limitations and insufficient virtual machine resources. These issues were investigated and resolved to provide a stable platform for the remainder of the project.

## Objectives
The objectives of this chapter were to:
- Deploy the Wazuh all in one platform.
- Configure a centralised security monitoring server.
- Verify successful communication between Wazuh services.
- Ensure the web dashboard was accessible.
- Optimise the server to support continuous endpoint monitoring.

# Infrastructure Challenges

Although the installation completed successfully, several infrastructure issues were encountered during deployment.

## Storage Limitation

The virtual machine was originally configured with a 28 GB virtual disk. As Wazuh indexed logs and created temporary vulnerability detection files, disk utilisation increased rapidly, eventually preventing the Indexer from operating correctly. Investigation identified that vulnerability detection temporary files occupied several gigabytes of storage.
To restore normal operation:
- unnecessary temporary files were removed
- vulnerability detection was temporarily disabled
- filesystem health was verified
- Wazuh services were restarted

## Virtual Machine Resource Upgrade
To provide sufficient resources for future endpoint monitoring, the virtual machine specifications were increased.

### Memory
The virtual machine memory was increased from **4 GB** to **8 GB** to provide additional capacity for the Wazuh Indexer.

### Storage
The virtual disk was expanded from **28 GB** to **60 GB**. After expanding the virtual disk, the Linux partition and Logical Volume Manager (LVM) configuration were resized to utilise the newly allocated storage. The following components were expanded:
- Partition
- Physical Volume (PV)
- Volume Group (VG)
- Logical Volume (LV)
- ext4 filesystem

Following expansion the server contained approximately **36 GB of free storage**, providing sufficient capacity for endpoint telemetry, log indexing and future attack simulations.

# Key Findings

- Successfully deployed the Wazuh all in one SIEM platform.
- Verified successful operation of the Manager, Indexer, Dashboard and Filebeat.
- Investigated and resolved storage limitations affecting the Indexer.
- Increased virtual machine resources from **4 GB RAM / 28 GB Storage** to **8 GB RAM / 60 GB Storage**.
- Expanded the Linux LVM filesystem to utilise the additional storage.
- Established a stable and scalable monitoring server ready for Windows and Linux endpoint integration.
